# Metrics

Метрики нужны не только для аналитики после релиза, но и для дизайна: они помогают понять, стала ли игра игрой.

## Северная звезда

**Доля игроков, которые завершили партию и начали вторую.**

Почему:

- завершение партии показывает, что loop понятен;
- вторая партия показывает, что есть реиграбельность;
- это важнее, чем просто клики по картам.

## Игровые метрики

| Метрика | Зачем нужна | Как читать |
|---|---|---|
| First card pick rate | Понял ли игрок, что делать | Низко = плохой onboarding |
| Turns completed | Доживает ли игрок до середины партии | Резкий обрыв = перегруз/скука |
| Finale reached | Доходит ли партия до результата | Низко = слишком длинно/непонятно |
| Replay rate | Хочется ли попробовать снова | Главный сигнал реиграбельности |
| Cards per session | Насколько активен игрок | Очень низко = интерфейс/темп мешает |
| Combo discovered rate | Видит ли игрок системные паттерны | Низко = комбо слишком скрыты |
| Debrief opened/read | Работает ли образовательная часть | Низко = разбор неинтересен/слишком длинный |

## Балансные метрики партии

| Метрика | Целевой смысл |
|---|---|
| Среднее число комбо за партию | 2–4 для короткой партии |
| Доля партий без комбо | Должна быть низкой после onboarding |
| Распределение финалов | Не должен доминировать один финал |
| Средний вред к финалу | Должен зависеть от стратегии |
| Средняя видимость к финалу | Должна расти от резких решений |
| Средняя автономия участников | Должна показывать человеческую цену |
| Уход/сопротивление участников | Должны быть заметными событиями, не шумом |

## Продуктовые метрики

| Метрика | Для чего |
|---|---|
| D1 retention | Возвращаются ли на следующий день |
| D7 retention | Есть ли долгосрочный интерес |
| Session length | Не перегрета ли партия |
| Sessions per user | Есть ли привычка |
| Ad completion rate | Если будет rewarded ad |
| Interstitial churn | Не убивает ли реклама желание играть |
| Scenario completion | Какие сценарии реально играют |

## Контентные метрики

| Метрика | Что показывает |
|---|---|
| Card pick frequency | Какие карты привлекательны |
| Card ignore frequency | Какие карты непонятны/скучны |
| Card regret signal | После каких карт игрок чаще бросает/откатывает |
| Red flag reveal frequency | Какие паттерны видны игроку |
| Debrief tag frequency | Какие темы чаще собираются |
| Participant focus | На каких персонажей смотрят |

## События аналитики v1

Пока аналитики нет, но события стоит проектировать заранее.

| Event | Когда отправлять | Поля |
|---|---|---|
| `game_start` | новая партия | scenarioId, saveVersion |
| `card_seen` | карта попала в руку | cardId, turn |
| `card_played` | карта выбрана | cardId, turn, groupMetrics |
| `combo_triggered` | сработало комбо | comboId, turn |
| `participant_selected` | открыт участник | participantId, turn |
| `finale_reached` | получен финал | finaleId, turn, score |
| `debrief_opened` | открыт разбор | finaleId, tags |
| `game_reset` | сброс партии | turn, scenarioId |

## Правило принятия решений

Если спорим о дизайне, сначала спрашиваем:

1. Это улучшает завершение первой партии?
2. Это увеличивает желание сыграть вторую?
3. Это делает последствия понятнее без лекции?
4. Это безопасно для платформ?
5. Это можно измерить?
