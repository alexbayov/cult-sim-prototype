# Backlog

## Как пользоваться

Каждая задача должна иметь:

- цель;
- почему важно;
- ожидаемый результат;
- критерий готовности.

Приоритеты:

- **P0** — нужно для playable vertical slice.
- **P1** — нужно для первой публикации на Яндекс Играх.
- **P2** — улучшает глубину/реиграбельность.
- **P3** — отложенные идеи.

## P0 — ближайшие задачи

### P0.0 Playable foundation

Цель: подготовить приложение к нормальной итерации.

Что сделать:

- localStorage save/load;
- версия save schema;
- сценарная структура контента;
- базовое описание art direction;
- manifest ассетов для портретов, карт и иконок.

Готово, когда:

- refresh страницы не сбрасывает текущую партию;
- есть единая точка сценария в коде;
- визуальные ассеты можно заказывать/генерировать по таблице.

### P0.1 Save/load

Цель: игрок не теряет партию и прогресс.

Что сделать:

- сохранить state в localStorage;
- добавить кнопку продолжить;
- добавить reset;
- добавить `saveVersion`.

Готово, когда:

- refresh страницы не сбрасывает партию;
- несовместимый save корректно игнорируется.

### P0.2 Контент в JSON

Цель: добавлять карты и участников без правки TypeScript.

Что сделать:

- вынести cards, participants и combos в JSON;
- добавить runtime validation или простую проверку структуры;
- оставить типы в TS;
- следующим шагом раздробить сценарий на `participants.json`, `cards.json`, `combos.json`, `finales.json`.

Готово, когда:

- новая карта добавляется через JSON;
- build ловит явные ошибки структуры.

### P0.3 Улучшить ход недели

Цель: сделать игру менее линейной.

Что сделать:

- добавить 2 действия за неделю;
- добавить разные типы слотов: практика, реакция, инфраструктура;
- добавить карты кризиса, на которые нужно отвечать;
- сделать руку карт не просто ротацией по порядку.

Готово, когда:

- игрок делает выбор, а не просто кликает все карты по очереди;
- кризисы меняют стратегию.

### P0.4 Финалы v2

Цель: финал должен ощущаться как итог стратегии.

Что сделать:

- 6–8 финалов;
- условия финалов в данных;
- финальный экран с разбором тегов, красных флагов и состояния участников.

Готово, когда:

- разные партии приводят к разным финалам;
- финал объясняет, почему он случился.

### P0.5 Мобильная адаптация

Цель: игра должна быть пригодна для Яндекс Игр и мобильного браузера.

Что сделать:

- mobile-first layout;
- нижняя панель карт;
- свернутые шкалы;
- крупные зоны нажатия.

Готово, когда:

- playable на 390×844;
- нет горизонтального скролла;
- ключевые действия доступны большим пальцем.

## P1 — Яндекс Игры

### P1.1 Yandex SDK adapter

Цель: не завязать игру жёстко на Яндекс.

Что сделать:

- `src/platform/yandex.ts`;
- `src/platform/storage.ts`;
- fallback для dev/local;
- флаг окружения.

Готово, когда:

- в dev работает localStorage;
- в Yandex build можно заменить storage на SDK.

### P1.2 Монетизация v1

Цель: аккуратно встроить рекламу.

Что сделать:

- interstitial после финала;
- rewarded ad за дополнительный архивный материал;
- не награждать за вредные действия.

Готово, когда:

- реклама не ломает партию;
- без рекламы игра остаётся полноценной.

### P1.3 Модерационный pass

Цель: снизить риск отказа платформ.

Что сделать:

- убрать инструктивные детали по криминалу/радикализации;
- тяжёлые темы показывать без способов и процедур;
- добавить дисклеймер: игра образовательная, не заменяет помощь специалиста.

Готово, когда:

- тексты безопасны для публичной web-платформы;
- нет конкретных инструкций для мошенничества, радикализации или самоповреждения.

## P2 — глубина игры

### P2.1 Новые сценарии

Кандидаты:

- псевдопсихологический тренинг;
- эзотерическая студия;
- корпоративная культура;
- конспирологический чат;
- крипто/скам-комьюнити.

### P2.2 Мета-прогрессия

Идеи:

- архив дел;
- словарь терминов;
- открытие новых оболочек;
- коллекция финалов;
- «разбор специалиста».

### P2.3 Состояния участников v2

Идеи:

- связи между участниками;
- семейные контакты;
- доверие к конкретному куратору;
- личные кризисы;
- индивидуальные пороги ухода/вербовки.

## P3 — отложенное

- Редактор карт внутри игры.
- Экспорт партии в отчёт.
- Режим «исследователь/следователь».
- Кооперация с психологом/куратором.
- Отдельная educational версия для лекций.
- Перенос на Godot/Unity, если web станет тесным.
