# Game Design — pivot к настоящей игре

> **Статус.** Design doc, фиксирующий продуктовый pivot после первого
> ручного плейтеста собранной main (post-#39). Плейтест показал что
> текущий «investigation/dossier» билд — это **интерактивный конспект**,
> а не игра: нет выбора (все 47 фрагментов можно положить в закладки), нет
> ошибки (каждый фрагмент подписан «связано с ...» и обозначает себя сам),
> нет цели (submit ничего не оценивает), нет последствий, нет
> реиграбельности. Wave 1-3 сделали игру **читабельной**. Этот документ
> описывает Wave 4 — как сделать её **играбельной**.
>
> **Этот файл — design only.** Никакого кода в этом PR. После approval —
> отдельный PR с прототипом vertical slice.

## 1. Premise

Игрок — **психолог-эксперт по работе с участниками деструктивных
сообществ**. Не «деструктолог», не «антисектант» — спокойный, измотанный,
компетентный частный специалист. К нему обращаются семьи и близкие людей,
которые «попали» куда-то и больше не возвращаются.

У героя есть **личный мотив**, который раскрывается постепенно по сезону
через мелочи (заметки в архиве, фоновые упоминания, выбор реплик в
интервью). Игрок не должен **читать** про его прошлое — он должен его
**заметить** к концу сезона. Default placeholder для имени в коде —
«Эксперт»; финальное имя/без-имени решим в писательском проходе.

Тон: серая компетентность, не пафос. Не «спасатель». Скорее
«разберём что у вас происходит, и подумаем что делать».

## 2. Структура сезона

**Independent cases, shared world (Variant X).** Каждый кейс —
отдельный запрос от отдельной семьи, отдельная группа, своя локация.
Между кейсами **нет общего антагониста** в Сезоне 1. Это сознательный
выбор: минимизировать писательскую работу до того как механика
доказала себя.

Если игра стрельнёт после Сезона 1 — Сезон 2 строится как
«клубок» (вариант Y) поверх уже отработанной механики, без её переделки.

Сезон 1 — 3 кейса. Vertical slice — **1 кейс** (см. §7).

## 3. Loop одного кейса

```txt
1. Брифинг                — короткое письмо/звонок от заявителя.
2. Стартовый стол         — 4-6 документов разных типов уже на столе.
3. Чтение + drag-highlight — игрок сам выделяет фразы из длинных текстов.
4. Workspace / блокнот    — выделенные фразы складываются в рабочие
                             наблюдения (player-named, не подписанные
                             заранее).
5. Action economy         — у игрока ограниченное число действий
                             (запрос интервью, перерыть архив, запросить
                             документы у клиента). 5-7 действий на кейс.
6. Gated NPC              — некоторые контакты раскрываются только если
                             игрок «предъявит» конкретное наблюдение
                             в момент диалога.
7. Финальный отчёт        — игрок выбирает 1 из 3-4 рекомендаций семье.
                             Это и есть ставка.
8. Эпилог через N месяцев — реальные последствия выбора. Не «оценка»,
                             а исход.
```

«Шум», «улики», «фрагменты», «паттерны» — **уйдут из UI**. Это
инженерный язык, он чужд игроку (подтверждено плейтестом). В коде
имена внутренних типов остаются (см. §6), но **в видимых строках их
быть не должно**. Валидатор `audit:visible-language` для этого и есть.

## 4. Core mechanics

### 4.1 Документы и drag-highlight

Каждый материал — это **длинный документ** одного из типов:

| Тип             | Пример контента                                                 |
|-----------------|-----------------------------------------------------------------|
| `chat`          | Скрин чата (плотный текст, диалоги, эмоджи)                     |
| `social`        | Пост в соцсети + комментарии под ним                            |
| `clipping`      | Газетная вырезка / онлайн-статья                                |
| `transcript`    | Расшифровка звонка / эфира                                      |
| `interview`     | Интервью которое герой берёт сам (диалог с выбором реплик)      |
| `document`      | Договор / расписка / справка / переписка по email               |
| `personal`      | Дневник / заметки участника                                     |

Контракт документа в коде (см. §6):

```ts
type Document = {
  id: string;
  type: 'chat' | 'social' | 'clipping' | 'transcript' | 'interview' | 'document' | 'personal';
  title: string;                  // что показывается в табе
  source: string;                 // откуда (для UI «источник: ...»)
  date?: string;                  // если важно для таймлайна
  body: string;                   // длинный связный текст
  keyPhrases: KeyPhrase[];        // скрытые «опоры», см. ниже
  defaultVisible: boolean;        // на старте или открывается действием
};

type KeyPhrase = {
  range: [start: number, end: number]; // позиция в body
  worksOn: string[];                   // hidden — на какие наблюдения работает
  weight: 'strong' | 'weak' | 'counter';
};
```

**Важно:** `keyPhrases` — **скрытая модель**. Игрок их не видит. Когда
игрок выделяет произвольный диапазон в body — система проверяет
пересекается ли с какой-то `KeyPhrase`. Если да — фраза «зацепилась».
Если нет — игрок просто положил в блокнот «нерелевантный» текст
(валидно, но не работает на финал).

Это означает: игрок может **ошибаться** (взять фон вместо опоры) и
может **не находить всё** (3 опоры из 5 — это всё ещё проходимо).

### 4.2 Workspace / блокнот

Правая колонка переименовывается в **«рабочий блокнот»**. Структура:

```txt
Гипотеза А: «здесь финансовое давление»
  ├─ выделенная фраза 1 (из чата с куратором)
  ├─ выделенная фраза 2 (из расписки)
  └─ выделенная фраза 3 (из интервью с матерью)

Гипотеза B: «здесь изоляция»
  └─ выделенная фраза 4 (из соцсети)

Под вопросом:
  └─ выделенная фраза 5 (не понял куда)
```

Гипотезы — **именованные слоты**, 3-4 на кейс. **Игрок их не придумывает
с нуля** в Сезоне 1 — слишком высокий cognitive load для онбординга.
Эксперт получает шаблонный лист «возможные рамки разбора», и **выбирает
какие из них использует в этом кейсе**. Сезон 2 — может появиться режим
free-form гипотез.

### 4.3 Action economy

Каждый кейс — фиксированный бюджет действий. Тип «действий»:

| Действие                                          | Стоимость | Что даёт                                                |
|---------------------------------------------------|-----------|---------------------------------------------------------|
| Прочитать стартовый документ                      | 0         | —                                                        |
| Перерыть архив СМИ по ключевому слову             | 1         | +1 `clipping`, иногда +1 `social`                       |
| Запросить интервью с контактом                    | 1         | +1 `interview` (если контакт согласился — см. 4.4)      |
| Запросить документы у клиента                     | 1         | +1 `document`                                            |
| «Сходить лично» (для финальных контактов)         | 2         | +1 `interview` повышенного веса, но риск отказа         |
| Сделать паузу / отчёт промежуточный               | 0         | —                                                        |

Бюджет — **5-7 действий** на кейс. Это создаёт **выбор**: какие линии
тянуть, какие игнорировать. Это и есть ядро игры.

### 4.4 Gated NPC

Контакты (NPC, которых можно интервьюировать) имеют **условие
раскрытия**:

```ts
type Contact = {
  id: string;
  name: string;
  role: string;                          // «бывшая участница», «местный журналист»
  initialState: 'public' | 'gated' | 'hostile' | 'unknown';
  gateRequirement?: {
    requiredNoteIds?: string[];          // нужно иметь в блокноте N
    requiredHypothesisFilled?: string;   // нужна минимум 1 опора в гипотезе X
    minWeight?: 'weak' | 'strong';
  };
  interview: InterviewTree;              // см. ниже
};
```

`public` — соглашается сразу. `gated` — соглашается только если в
блокноте уже есть достаточно. `hostile` — соглашается только после
второй попытки + конкретного предъявления. `unknown` — контакт ещё не
найден; появляется после `clipping` или другого документа.

**`interview`** — мини-дерево диалога (3-5 узлов), где у игрока есть
2-3 варианта реплики. Реплики могут разблокировать новые куски
интервью **если** игрок «предъявил» конкретную фразу из блокнота.

Это и есть твоя идея «показал что нашёл → собеседник раскалывается».

### 4.5 Финальный отчёт и эпилог

В момент когда игрок считает что готов (или закончились действия) —
кнопка **«Сформулировать отчёт семье»**. Открывается экран:

```txt
Семье: [Иванова Е.]
Подопечный: [Сергей И., 27]

Ваша рекомендация:
( ) A. Активное вмешательство сейчас опасно — переждать, держать тёплый
       контакт через письма, попытка встречи через 6 месяцев.
( ) B. Финансовое давление приближается к точке невозврата. Юридическая
       консультация + срочная очная попытка контакта по подготовленному
       тексту (мы дадим).
( ) C. Это его выбор, не группа. Вы остаётесь рядом, если когда вернётся.

(O) Я ещё не готов — вернуться к материалам
```

3-4 рекомендации. Каждая имеет **скрытый wantsHypotheses-набор** — какие
гипотезы должны быть подкреплены и в какой степени. Игрок выбирает —
система считает совпадение — **эпилог** через 3/6/12 месяцев. У каждой
рекомендации — 2-3 вариации эпилога (точная / неточная / случайная
ошибка), чтобы выбор не был чёрно-белым «угадал/не угадал».

Эпилог = **2-3 короткие сцены** (текст + лёгкий иконографический сигнал
без оценочных эмоций). Не «вы выиграли». «Вот что вышло».

## 5. Концепт первого кейса (vertical slice)

**Кейс-01: «Прорыв» — Сергей.**

Заявитель: мать Сергея Иванова, 56 лет, бухгалтер из Подмосковья.
Подопечный: Сергей, 27, программист.

Полтора года назад прошёл марафон личной эффективности «Прорыв».
Сначала радовался — «нашёл свою команду». Полгода назад уволился с
работы, переехал в загородный коттедж к «наставнику» Леониду Б. Месяц
назад продал машину. Не звонит. Деньги переводит куда-то. Мать пишет
эксперту через сайт центра помощи семьям.

**Стартовый стол (4 документа):**

1. `personal` — переписка матери с сыном за последние 3 месяца
   (постепенно холодеет, реплики сына становятся как будто
   копированными).
2. `chat` — скрин чата «Прорыв · Близкий круг» с участием Сергея и
   куратора (полный плотный текст, ~40 сообщений, разное).
3. `social` — пост Сергея в соцсети 4 месяца назад («встретил людей,
   которые наконец-то понимают»), + 12 комментариев под ним.
4. `clipping` — короткая статья 6-летней давности из регионального
   СМИ: «Бизнес-тренинги под руководством Леонида Б.: пострадавшие
   обратились в полицию» (без приговора, дело прекращено).

**Контакты (3 NPC):**

- **Мать Сергея** — `public`, всегда соглашается. Базовая информация.
- **Бывший участник «Прорыва» (Денис К.)** — `gated`. Заговорит, только
  если в блокноте есть **минимум 1 weak-опора** по гипотезе «финансовое
  давление». До этого: «ничего особенного не было, мы просто учились».
- **Журналист местного СМИ (Анна П.)** — `gated`. Заговорит, если игрок
  предъявил `clipping`. Тогда: «у меня есть ещё материалы из той
  истории — могу прислать».

**Действия (бюджет: 6):**

- Перерыть архив СМИ → +1-2 `clipping` про Леонида Б.
- Запросить интервью с матерью → +1 `interview`.
- Запросить интервью с Денисом К. (если разблокирован).
- Запросить интервью с Анной П. (если разблокирована).
- Запросить документы у клиента → переписка по email + копия договора
  Сергея с центром.
- «Сходить лично» к Леониду Б. → высокий риск отказа, но если
  получилось — мощный `interview` с ним самим.

**3 финальные рекомендации, каждая с 2-3 вариантами эпилога:**

- **A. Переждать + тёплый контакт.** Правильно если: в блокноте
  достаточно weak-опор без strong-опор. Эпилог-правильно: через 8
  месяцев Сергей сам пишет матери. Эпилог-неточно: ничего не происходит,
  семья теряет контакт окончательно.
- **B. Срочное юридическое + очный контакт.** Правильно если: есть
  strong-опора по финансовому давлению и есть гипотеза изоляции. Эпилог-
  правильно: юрист помогает, Сергей через 3 месяца сам уходит. Эпилог-
  неточно (слишком рано) — Сергей оборвал контакт навсегда.
- **C. Это его выбор.** Правильно если: блокнот пустой или гипотез нет.
  Эпилог-правильно: редкие звонки, через 2 года стабильная переписка.
  Эпилог-неточно (когда на самом деле надо было B): через год Сергей
  банкрот, депрессия, контакт восстанавливается тяжело.

## 6. Что остаётся / что переделывается

### Остаётся как есть

- React + TypeScript + Vite стек.
- `localStorage` persist (progress + achievements per case).
- Yandex SDK adapter и `build:yandex`.
- `validate:investigation` валидатор и `audit:visible-language` (теперь
  ещё критичнее).
- Структура папок `src/game/cases/<case-id>/` под JSON-контент.
- Сезонная оболочка `src/game/seasons/season-01.json`.
- Типы `Person` (теперь играют роль `Contact`).
- Подход «один кейс — отдельная папка JSON-ов» (просто JSON-ы другой
  формы).

### Переделывается

- **`Evidence[]` (47 фрагментов с подписанным `pattern_id`)** → удаляется
  как видимая сущность. Заменяется на `Document.body` + `Document.keyPhrases`.
- **`Pattern[]` (12 готовых наблюдений)** → остаётся как **внутренний
  словарь** для матчинга keyPhrases, но **не показывается игроку как
  готовый список**. Игрок видит «гипотезы» (3-4 шаблонных слота),
  заполняет их сам.
- **`bookmarkedFragmentIds`** → заменяется на `notebookEntries[]`
  (выделенные диапазоны текста + к какой гипотезе игрок их положил).
- **`Report.outcomes` (6 готовых исходов)** → заменяется на
  `Recommendation[]` (3-4 на кейс) + `Epilogue[]` (множественные
  вариации).
- **`DossierApp.tsx`** — большая переделка. Из «3 колонки» в
  workspace: документы / блокнот / контакты / действия. Возможно с
  переключением вкладок на 390×844.
- **«Шум», «улики», «фрагменты», «паттерны»** — уходят из любой видимой
  копии. Visible-language deny list пополняется этими терминами.

### Удаляется

- `src/investigation/resolutionModel.ts` в текущем виде — переделывается
  в `recommendationModel.ts` с другой логикой.
- Текущая структура `report.json` / `debrief.json` — переделывается в
  `recommendations.json` + `epilogues.json`.
- Текущие 2 кейса (`info-business-marathon`, `family-retreat-center`)
  переходят в **архив**. Файлы остаются для референса, но через
  `season-01.json` не загружаются.

## 7. Vertical slice scope (Wave 4 = pivot)

**Цель:** через **7-9 рабочих дней одним последовательным потоком**
получить playable proof:

- 1 новый кейс по шаблону `case-01` из §5.
- 4 стартовых документа (1 personal + 1 chat + 1 social + 1 clipping).
- 3 контакта (1 public, 2 gated).
- 6 действий бюджет.
- Drag-highlight в любом документе.
- Notebook с 3 гипотезами.
- 3 рекомендации, 6-9 вариантов эпилога.
- Mobile 390×844 — не сломать.

**Что НЕ входит в vertical slice:**

- Второй и третий кейсы сезона.
- Free-form гипотезы.
- Аудио/видео плеер для `transcript` — текстовая расшифровка.
- Тайм-таймер / dynamic ивенты по дням.
- Многоязычность.
- Animations / переходы кроме базовых CSS.

**Дисциплина:**

- **Один поток**, не параллельные Devin'ы. Pivot ломает базовые типы —
  параллельный мерж конфликтует на 90% файлов.
- Сначала data-модель + JSON-схема первого кейса. Потом UI поверх. Потом
  контент-полишинг.
- После каждого PR — играем сами на main, фиксируем impressions
  в `docs/PLAYTEST_NOTES.md`.

## 8. Open questions (требуют решения до vertical slice кода)

1. **Имя героя:** «Эксперт» (default placeholder, нейтрально), своё имя,
   или второе лицо без имени («вы»)?
2. **3 шаблонных гипотезы для кейса-01** — какие именно (рабочие
   названия: «финансовое давление», «изоляция от близких», «зависимость
   от наставника», «обращение / новая идентичность»)? Можем закрепить в
   следующем PR.
3. **Drag-highlight UX на мобильном 390×844** — long-press → range
   selection — это сложный UX на сенсорном. Возможно нужен fallback
   «выбрать абзац целиком», или предварительно подсвеченные **areas**
   которые игрок просто тапает (упрощение, но менее tactile). Решаем
   когда будем прототипировать.
4. **Эпилог-вариация** — рандомный или детерминированный по подобию
   совпадения? Скорее всего детерминированный, чтобы reset → тот же
   результат → проще учиться.
5. **Сохранение прогресса в середине кейса** — да, через `localStorage`,
   но детали ключей в момент кода.

Все эти вопросы — не блокеры для design doc. Они блокеры для **кода**.
